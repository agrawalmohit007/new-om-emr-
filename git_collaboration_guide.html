Git Team Collaboration Guide

This document describes the exact Git-based SCM workflow for coordinating frontend UI features (using Antigravity on Windows) and backend database logic (on macOS).

1. SCM Branch Strategy

To ensure code remains clean and unified, both developers commit to specialized branches, combining updates via Pull Requests:

main Branch
Stable & Unified
↔
ui-features Branch
Frontend / UI Widgets
↔
backend-storage Branch
Database / Schema Setup
  • main: The stable core. Serves as the unified application deployed to testing/production.
  • ui-features (Your branch): Where Antigravity pushes layout changes and Frontend updates.
  • backend-storage (Backend branch): Where the backend developer manages database tables, schemas, and API routes.

2. macOS Backend Developer Setup

The backend developer executes the following setup commands on their Mac:

# 1. Clone the repository
git clone https://github.com/agrawalmohit007/new-om-emr-.git
cd new-om-emr-

# 2. Install dependencies & configure local environment
npm install
cp .env.example .env

# 3. Create and push the backend branch
git checkout -b backend-storage
git add .
git commit -m "Implement data storage and database integration"
git push -u origin backend-storage

3. Your Daily Frontend Workflow (with Antigravity)

When you request UI additions or layout adjustments, Antigravity automates these Git steps:

# 1. Pull the backend updates from main
git pull origin main

# 2. Code updates and compile verification

# 3. Commit and push to UI branch
git checkout -b ui-features
git add .
git commit -m "UI: Complete EMR layout and autocompletions"
git push origin ui-features

💡 Tip: Merges will automatically execute successfully on GitHub since frontend design and backend database schemas reside in separate code files.
